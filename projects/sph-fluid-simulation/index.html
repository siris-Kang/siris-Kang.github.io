SPH Fluid Simulation

CUDA 기반 SPH 유체 시뮬레이션 구현 프로젝트

Source code

1. 프로젝트 개요

  • CUDA를 이용해 입자 기반 유체의 움직임을 구현한 프로젝트
  • SPH(Smoothed Particle Hydrodynamics) 방식으로 밀도, 압력, 점성, 외력을 계산해 유체를 시뮬레이션하는 것이 목표
  • 계산 결과를 Unreal과 연동해 particle의 움직임을 시각적으로 표현

주요 기능

  • 입자 기반 유체 시뮬레이션 구현
  • 밀도, 압력, 점성, 외력 계산 로직 구성
  • 시간에 따른 속도 및 위치 업데이트
  • Unreal 환경에서 시뮬레이션 결과 시각화

2. 담당 역할

  • CUDA를 이용한 SPH 유체 시뮬레이션 구현
  • 커널 함수 기반 밀도 계산 로직 구현
  • 압력, 점성, 외력 계산을 통한 입자 힘 반영
  • 가속도, 속도, 위치 업데이트 로직 구현
  • Unreal과 연동해 시뮬레이션 결과 시각화

3. 사용 기술

CUDA

  • 입자 수가 많은 유체 시뮬레이션을 병렬로 계산하기 위해 사용
  • 밀도, 압력, 점성, 외력 계산을 GPU에서 처리하는 구조 직접 구현

SPH (Smoothed Particle Hydrodynamics)

  • 격자 기반이 아니라 입자 기반으로 유체를 표현하기 위해 선택
  • 각 입자 간 상호작용을 통해 유체 움직임을 계산하는 방식이 프로젝트 목표와 부합

Unreal Engine

  • 시뮬레이션 결과를 시각적으로 확인하고 동작을 직관적으로 검증하기 위해 사용
  • 계산 결과를 단순 수치가 아닌 실제 화면으로 확인하며 디버깅에 활용

4. 구현 사항

  • SPH 수식을 기반으로 입자의 밀도와 압력을 계산하는 커널 로직 구현
  • 점성 및 외력을 반영해 각 입자에 작용하는 힘 계산
  • 가속도, 속도, 위치를 시간에 따라 업데이트하는 시뮬레이션 흐름 구성
  • 계산된 particle 움직임을 Unreal과 연동해 시각적으로 표현

5. 문제 해결 사례

1) Boundary를 적용해도 particle이 경계를 뚫고 나가던 문제

상황

boundary를 구성해 입자가 밖으로 새지 않도록 설계했지만, 시뮬레이션 중 particle이 계속 경계를 뚫고 나가는 문제가 발생했다.

해결

  • 바운더리 오브젝트를 실제 삼각형 메시가 아닌 회전 가능한 박스(OBB)로 근사해 충돌 검사
  • 파티클이 박스 내부로 들어오면 가장 가까운 면 밖으로 밀어내고, 해당 면 안쪽으로 향하는 속도 성분만 제거

성과

  • 파티클이 경계를 통과하던 문제를 기하 연산 기반 충돌 처리로 완화
  • Unreal의 좌표계와 시뮬레이션 좌표계를 변환해 CUDA 충돌 계산에 반영

의미

  • 물리 시뮬레이션에서는 복잡한 메시 충돌보다 문제에 맞는 단순화된 충돌 모델 설계가 중요하다는 점 확인

6. 성과 및 결과

  • CUDA 기반 SPH 유체 시뮬레이션과 Unreal 시각화 흐름 직접 구현
  • 입자 기반 물리 시뮬레이션을 GPU 병렬 연산으로 다루는 경험 확보
  • 경계 처리와 수치 안정성의 중요성을 실제 구현 과정에서 학습
