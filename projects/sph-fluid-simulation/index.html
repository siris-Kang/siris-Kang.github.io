SPH Fluid Simulation

CUDA 기반 SPH 유체 시뮬레이션 구현 프로젝트

1. 프로젝트 개요

  • CUDA를 이용해 입자 기반 유체의 움직임을 구현한 프로젝트입니다.
  • SPH(Smoothed Particle Hydrodynamics) 방식으로 밀도, 압력, 점성, 외력을 계산해 유체를 시뮬레이션하는 것이 목표였습니다.
  • 계산 결과를 Unreal과 연동해 particle의 움직임을 시각적으로 표현했습니다.

주요 기능

  • 입자 기반 유체 시뮬레이션 구현
  • 밀도, 압력, 점성, 외력 계산 로직 구성
  • 시간에 따른 속도 및 위치 업데이트
  • Unreal 환경에서 시뮬레이션 결과 시각화

2. 담당 역할

  • CUDA를 이용한 SPH 유체 시뮬레이션 구현
  • 커널 함수 기반 밀도 계산 로직 구현
  • 압력, 점성, 외력 계산을 통한 입자 힘 반영
  • 가속도, 속도, 위치 업데이트 로직 구현
  • Unreal과 연동해 시뮬레이션 결과 시각화

3. 사용 기술

CUDA

  • 입자 수가 많은 유체 시뮬레이션을 병렬로 계산하기 위해 사용했습니다.
  • 밀도, 압력, 점성, 외력 계산을 GPU에서 처리하는 구조를 직접 구현해볼 수 있었습니다.

SPH (Smoothed Particle Hydrodynamics)

  • 격자 기반이 아니라 입자 기반으로 유체를 표현하기 위해 선택했습니다.
  • 각 입자 간 상호작용을 통해 유체 움직임을 계산하는 방식이 프로젝트 목표와 잘 맞았습니다.

Unreal Engine

  • 시뮬레이션 결과를 시각적으로 확인하고 동작을 직관적으로 검증하기 위해 사용했습니다.
  • 계산 결과를 단순 수치가 아닌 실제 화면으로 확인하면서 디버깅하기 좋았습니다.

4. 구현 사항

  • SPH 수식을 기반으로 입자의 밀도와 압력을 계산하는 커널 로직을 구현했습니다.
  • 점성 및 외력을 반영해 각 입자에 작용하는 힘을 계산했습니다.
  • 가속도, 속도, 위치를 시간에 따라 업데이트하는 시뮬레이션 흐름을 구성했습니다.
  • 계산된 particle 움직임을 Unreal과 연동해 시각적으로 표현했습니다.

5. 문제 해결 사례

1) Boundary를 적용해도 particle이 경계를 뚫고 나가던 문제

상황

boundary를 구성해 입자가 밖으로 새지 않도록 설계했지만, 시뮬레이션 중 particle이 계속 경계를 뚫고 나가는 문제가 발생했습니다.

해결

  • 경계 처리 방식과 충돌 반응 계산 중 어느 쪽이 더 큰 영향을 주는지 먼저 분리해 보려 했습니다.
  • SPH 파라미터, 경계 조건, 시간 간격 설정을 함께 조정하며 문제 범위를 좁혔습니다.
  • 프로젝트 기간과 GPU 환경 제약으로 완전한 해결에는 이르지 못했지만, 문제 지점을 경계 조건과 수치 안정성 이슈로 정리했습니다.

성과

  • 미해결 이슈를 막연한 오류가 아니라 경계 처리와 수치 안정성 문제로 구체화했습니다.
  • 추가 실험이 필요한 지점을 명확히 정리해 후속 개선 방향을 도출했습니다.

의미

  • 물리 시뮬레이션에서는 기능 구현만큼 경계 조건과 수치 안정성 설계가 중요하다는 점을 체감했습니다.
  • 미해결 이슈가 남았지만, CUDA 기반 병렬 연산과 시뮬레이션 안정성 문제를 직접 고민해본 경험을 얻었습니다.

6. 성과 및 결과

  • CUDA 기반 SPH 유체 시뮬레이션과 Unreal 시각화 흐름을 직접 구현했습니다.
  • 입자 기반 물리 시뮬레이션을 GPU 병렬 연산으로 다루는 경험을 쌓았습니다.
  • 경계 처리와 수치 안정성의 중요성을 실제 구현 과정에서 학습했습니다.
