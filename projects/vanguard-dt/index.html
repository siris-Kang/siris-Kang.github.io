VANGUARD-DT

RC카 기반 임베디드 제어와 MQTT 연동 프로젝트

0. 프로젝트 소개

VANGUARD-DT 소개 영상

VANGUARD-DT GitHub

1. 프로젝트 개요

  • RC카 기반 시스템을 직접 구성하며 임베디드 제어와 통신 구조를 함께 경험한 프로젝트입니다.
  • 상위 시스템과의 연동을 위해 MQTT 통신을 적용하고, Qt를 활용해 제어 및 상태 확인용 인터페이스를 구성했습니다.
  • STM32 NUCLEO-F103RB를 중심으로 모터와 서보를 제어하며, 제어부와 통신부가 함께 동작하는 테스트베드를 만드는 것이 목표였습니다.

주요 기능

  • STM32 기반 PWM 모터 제어와 UART 명령 수신
  • MQTT 기반 상위 시스템 연동
  • Qt GUI를 통한 상태 확인과 명령 입력
  • 제어와 통신이 연결된 RC카 프로토타입 구성

2. 담당 역할

  • STM32CubeIDE 환경 설정과 PWM 기반 모터 제어 구현
  • UART 기반 명령 수신 구조 설계 및 서보 제어 반영
  • MQTT 메시지 송수신 흐름 정리와 기본 통신 로직 구현
  • Qt 기반 제어 UI 구성과 상태 확인 화면 구현

3. 사용 기술

STM32CubeIDE

  • STM32 NUCLEO-F103RB의 핀 설정과 주변장치 초기화를 직접 구성하기 위해 사용했습니다.
  • .ioc 기반 설정을 통해 PWM, UART 등 제어에 필요한 자원을 안정적으로 관리할 수 있었습니다.

UART

  • 상위 시스템이 보내는 속도 및 조향 명령을 STM32가 바로 수신해 반영하기 위해 선택했습니다.
  • 하드웨어 제어부와 상위 보드 사이를 단순하고 명확하게 연결할 수 있는 구조였습니다.

MQTT

  • 상위 시스템과 상태/명령 데이터를 주고받는 메시지 기반 구조가 필요해 적용했습니다.
  • 토픽 단위로 송수신 역할을 분리할 수 있어 제어 흐름을 정리하기 좋았습니다.

Qt

  • RC카 상태를 한 화면에서 보고 명령을 입력할 수 있는 데스크톱 UI를 빠르게 구성하기 위해 사용했습니다.
  • 통신과 제어 결과를 시각적으로 확인하며 디버깅하기에도 적합했습니다.

4. 구현 사항

  • STM32CubeIDE에서 PWM 채널과 UART 수신 구조를 설정하고 DC 모터와 서보를 제어했습니다.
  • 외부 명령을 UART로 받아 속도와 조향 값에 반영하는 제어 로직을 구성했습니다.
  • MQTT 토픽 구조를 정리해 상위 시스템과의 기본 송수신 흐름을 구현했습니다.
  • Qt 기반 화면에서 상태를 확인하고 테스트 명령을 입력할 수 있는 UI를 구성했습니다.

5. 문제 해결 사례

1) STM32 기반 모터/서보 제어가 의도대로 동작하지 않던 문제

상황

라즈베리파이와 STM32가 UART로 주고받는 데이터 형식이 맞지 않아, 서보가 의도와 다르게 움직이는 문제가 발생했습니다.

해결

  • 통신 자체 문제인지, STM32가 수신 값을 잘못 해석하는 문제인지 먼저 분리해 확인했습니다.
  • 송신 측 데이터 형식과 수신 측 해석 순서, 범위를 비교하며 명령 구조를 단순화했습니다.
  • UART 수신 테스트와 고정값 서보 테스트를 분리해 진행한 뒤, 범위를 제한한 값을 제어 로직에 반영하도록 수정했습니다.

성과

  • 외부 명령에 따라 서보가 안정적으로 반응하는 구조를 만들었습니다.
  • 통신 문제와 제어 문제를 분리해 디버깅하는 기준을 확보했습니다.

의미

  • 임베디드 제어에서는 하드웨어 동작 이상이 항상 코드 문제만은 아니라는 점을 체감했습니다.
  • 통신 형식과 제어 로직을 분리해 검증하는 방식이 이후 디버깅 기준이 되었습니다.

2) ROS와 MQTT 연동 구조가 혼란스럽던 문제

상황

ROS와 MQTT 사이에서 어떤 데이터를 어떤 흐름으로 주고받아야 하는지 구조가 명확하지 않아 메시지 연동 과정이 복잡하게 느껴졌습니다.

해결

  • ROS 내부 토픽 흐름과 MQTT 브로커 기반 메시지 흐름을 먼저 분리해 이해했습니다.
  • ROS → MQTT → UI/제어 로직 순서로 데이터 흐름을 단순하게 정리했습니다.
  • 송신 메시지와 수신 메시지의 역할을 topic별로 나누고, MQTT 단독 테스트 후 ROS와 단계적으로 연결했습니다.

성과

  • 시스템 상태 확인과 명령 전달이 가능한 기본 통신 구조를 구현했습니다.
  • 데이터 흐름이 정리되면서 UI와 제어 로직을 함께 테스트하기 쉬워졌습니다.

의미

  • 통신 문제는 코드 한 부분보다 전체 데이터 흐름 관점에서 정리해야 한다는 기준을 얻었습니다.
  • 복합 시스템에서는 기능 구현보다 먼저 메시지 구조를 명확히 잡는 일이 중요하다는 점을 확인했습니다.

6. 성과 및 결과

  • STM32 제어부와 MQTT 통신부, Qt UI를 연결한 RC카 기반 임베디드 테스트베드를 완성했습니다.
  • 모터와 서보 제어, 명령 수신, 상태 확인이 이어지는 기본 시스템 흐름을 직접 구성했습니다.
  • 제어 로직과 통신 구조를 함께 다루며 임베디드 시스템 통합 경험을 쌓았습니다.
